# Część 2 — Logika (Backend)

## Technologia

- **Język:** Node.js + Express
- **Baza danych:** PostgreSQL
- **ORM:** Prisma lub Sequelize
- **Generowanie PDF:** PDFKit (Node) lub pdfmake

## Endpointy API

| Endpoint | Co robi |
|----------|---------|
| `POST /auth/register` | Tworzy nowe konto. Przyjmuje: email, hasło, nazwa firmy. Zwraca: token JWT. |
| `POST /auth/login` | Loguje użytkownika. Przyjmuje: email, hasło. Zwraca: token JWT. |
| `POST /documents/generate` | Główny endpoint. Przyjmuje dane z formularza. Uruchamia logikę doboru dokumentów. Generuje PDF-y. Zwraca linki do pobrania. |
| `GET /documents/history` | Zwraca historię wygenerowanych dokumentów dla zalogowanego użytkownika. |
| `GET /documents/:id/download` | Pobiera konkretny wygenerowany plik PDF. |
| `POST /companies` | Zapisuje nową firmę (nadawca/odbiorca) do bazy użytkownika. |
| `GET /companies` | Zwraca listę zapisanych firm użytkownika. |
| `PUT /companies/:id` | Aktualizuje dane firmy. |
| `DELETE /companies/:id` | Usuwa firmę z bazy. |
| `GET /subscription/status` | Sprawdza aktualny plan, datę odnowienia, limit dokumentów. |
| `POST /subscription/create` | Tworzy subskrypcję przez Stripe. |
| `POST /webhook/stripe` | Odbiera zdarzenia od Stripe (płatność, anulowanie). |

## Główna logika — dobór dokumentów

### Krok 1 — Określenie zestawu dokumentów

| Typ transportu | Trasa | Dokumenty |
|---------------|-------|-----------|
| Drogowy (TIR) | W obrębie UE | CMR, Packing List, Faktura handlowa, Zlecenie transportowe |
| Drogowy (TIR) | Poza UE (np. PL→CN) | CMR, Packing List, Faktura handlowa, Zlecenie transportowe + Faktura proforma, Świadectwo pochodzenia, SAD |
| Morski | W obrębie UE | Sea Waybill, Booking Confirmation, Packing List morski |
| Morski | Poza UE (np. PL→US) | Sea Waybill, Booking Confirmation, Packing List morski + Faktura proforma, Świadectwo pochodzenia, SAD |

### Krok 2 — Wypełnienie szablonów
- System bierze szablon każdego dokumentu (np. CMR_template.json)
- Podstawia dane użytkownika w miejsce zmiennych: `{{nadawca}}`, `{{odbiorca}}`, `{{towar}}` itd.
- Generuje gotowy dokument jako obiekt JSON z uzupełnionymi polami

### Krok 3 — Generowanie PDF
- Każdy wypełniony dokument jest przekazany do biblioteki PDFKit
- PDFKit rysuje układ dokumentu (tabele, pola, podpisy) i generuje plik PDF
- Pliki są tymczasowo zapisywane na serwerze lub w cloud storage (np. AWS S3)
- Użytkownik dostaje linki do pobrania — ważne przez 24h

### Krok 4 — Zapis do historii
- Zestaw dokumentów jest zapisywany w bazie danych przypisany do konta użytkownika
- Zapisywane są: data, trasa, typ transportu, lista dokumentów, linki do plików

## Autoryzacja i bezpieczeństwo

| Mechanizm | Opis |
|-----------|------|
| JWT Token | Po zalogowaniu użytkownik dostaje token. Każde zapytanie do API musi go zawierać w nagłówku. |
| Hasła | Przechowywane jako hash (bcrypt) — nigdy jako czysty tekst. |
| Rate limiting | Limit zapytań do API — ochrona przed atakami (np. max 100 zapytań/minutę). |
| Walidacja danych | Każde dane z formularza są walidowane po stronie backendu. |
| HTTPS | Cały ruch szyfrowany. |

## Płatności (Stripe)
- Stripe obsługuje abonamenty miesięczne i płatności jednorazowe za dokument
- Backend nasłuchuje webhooków od Stripe
- Po potwierdzeniu płatności — aktualizacja statusu konta w bazie
- Przy generowaniu dokumentu system sprawdza czy użytkownik ma aktywny plan

## Pola dokumentów

### CMR — List przewozowy drogowy (24 pola)

| Pole | Zawartość |
|------|-----------|
| Pole 1 | Nadawca: nazwa, adres, kraj |
| Pole 2 | Odbiorca: nazwa, adres, kraj |
| Pole 3 | Miejsce dostawy (adres) |
| Pole 4 | Miejsce i data załadunku |
| Pole 5 | Dokumenty dołączone (faktury, SAD) |
| Pole 6 | Znaki i numery przesyłki |
| Pole 7 | Liczba opakowań |
| Pole 8 | Rodzaj opakowania |
| Pole 9 | Nazwa towaru |
| Pole 10 | Numer statystyczny (HS code) |
| Pole 11 | Waga brutto w kg |
| Pole 12 | Objętość w m3 |
| Pole 13 | Instrukcje nadawcy |
| Pole 14 | Klasa ADR (towary niebezpieczne) |
| Pole 15 | Warunki zapłaty frachtu |
| Pole 16 | Przewoźnik: nazwa i adres |
| Pole 17 | Kolejny przewoźnik (opcjonalne) |
| Pole 18 | Zastrzeżenia i uwagi przewoźnika |
| Pole 19 | Postanowienia specjalne |
| Pole 20 | Do zapłaty przez nadawcę |
| Pole 21 | Wystawiono w (miasto), data |
| Pole 22 | Podpis nadawcy |
| Pole 23 | Podpis przewoźnika |
| Pole 24 | Podpis odbiorcy + data dostawy |

### Packing List

| Pole | Zawartość |
|------|-----------|
| Nagłówek | Numer Packing List, data wystawienia |
| Nadawca / Exporter | Nazwa firmy, adres, kraj, VAT |
| Odbiorca / Consignee | Nazwa firmy, adres, kraj, VAT |
| Numer faktury | Referencja do faktury handlowej |
| Kraj pochodzenia | Gdzie towar był wyprodukowany |
| Kraj przeznaczenia | Dokąd jedzie |
| Pozycje (tabela) | Nr poz. / Opis / Ilość / Waga netto / Waga brutto / Wymiary / Objętość |
| Podsumowanie | Łączna liczba paczek, łączna waga, łączna objętość |
| Podpis | Podpis i pieczęć nadawcy |

### Faktura handlowa (Commercial Invoice)

| Pole | Zawartość |
|------|-----------|
| Numer faktury | Unikalny numer wystawiany przez nadawcę |
| Data wystawienia | Data faktury |
| Sprzedawca (Seller) | Pełne dane firmy, VAT, bank |
| Kupujący (Buyer) | Pełne dane firmy, VAT |
| Warunki dostawy | Incoterms: EXW, DAP, CIF itp. |
| Warunki płatności | np. 30 dni od wystawienia |
| Waluta | EUR / CHF / USD |
| Pozycje (tabela) | Nr / Opis towaru / Kod HS / Ilość / Cena jedn. / Wartość |
| Podsumowanie | Wartość netto, podatki, wartość brutto |
| Bank do przelewu | IBAN, BIC/SWIFT, nazwa banku |
| Podpis | Podpis i pieczęć sprzedawcy |

### SAD — Deklaracja celna

| Pole | Zawartość |
|------|-----------|
| Pole 1 | Typ deklaracji: import (IM) / eksport (EX) |
| Pole 2 | Nadawca / Eksporter |
| Pole 8 | Odbiorca / Importer |
| Pole 15 | Kraj wysyłki |
| Pole 17 | Kraj przeznaczenia |
| Pole 22 | Waluta i łączna wartość faktury |
| Pole 31 | Opis towaru, znaki, numery, ilość i rodzaj opakowań |
| Pole 33 | Kod towaru (kod CN / HS) |
| Pole 35 | Masa brutto w kg |
| Pole 44 | Dodatkowe informacje, załączone dokumenty |

### Sea Waybill

| Pole | Zawartość |
|------|-----------|
| Shipper (Nadawca) | Pełne dane firmy eksportującej |
| Consignee (Odbiorca) | Pełne dane firmy importującej |
| Notify Party | Kto ma być powiadomiony przy przybyciu statku |
| Vessel / Voyage | Nazwa statku i numer rejsu |
| Port of Loading | Port załadunku |
| Port of Discharge | Port rozładunku |
| Container No. | Numer kontenera i numer plomby |
| Description of Goods | Opis towaru |
| Gross Weight | Waga brutto w kg |
| Measurement | Objętość w m3 |

### Zlecenie transportowe

| Pole | Zawartość |
|------|-----------|
| Numer zlecenia | Unikalny numer wewnętrzny |
| Zleceniodawca | Firma zlecająca transport (nadawca) |
| Przewoźnik | Firma realizująca transport |
| Miejsce załadunku | Adres i data załadunku |
| Miejsce rozładunku | Adres i planowana data rozładunku |
| Opis towaru | Co jedzie, waga, palety |
| Fracht | Uzgodniona kwota za przewóz |
| Warunki płatności | Termin i forma płatności |
| Podpisy | Podpis zleceniodawcy i potwierdzenie przewoźnika |
