# Część 3 — Baza danych

## Technologia

- **System:** PostgreSQL
- **ORM:** Prisma lub Sequelize
- **Hosting:** Supabase / Railway / Neon (cloud PostgreSQL)

## Tabele

### users (użytkownicy)

| Kolumna | Opis |
|---------|------|
| id | UUID — unikalny identyfikator użytkownika |
| email | Adres email — używany do logowania, musi być unikalny |
| password_hash | Hasło zaszyfrowane bcrypt — nigdy czysty tekst |
| company_name | Nazwa firmy użytkownika |
| vat_number | NIP / VAT (opcjonalne) |
| plan | Aktualny plan: free / monthly / pay_per_doc |
| plan_expires_at | Data wygaśnięcia abonamentu miesięcznego |
| stripe_customer_id | ID klienta w Stripe — do obsługi płatności |
| created_at | Data rejestracji |

### companies (zapisane firmy)

| Kolumna | Opis |
|---------|------|
| id | UUID |
| user_id | Powiązanie z tabelą users — kto dodał tę firmę |
| name | Nazwa firmy (np. ABC Transport GmbH) |
| address | Ulica i numer |
| city | Miasto |
| postal_code | Kod pocztowy |
| country | Kraj (kod ISO, np. PL, DE, CN) |
| vat_number | NIP / numer VAT |
| type | Rola firmy: sender / receiver / both |
| created_at | Data dodania |

### document_sets (zestawy wygenerowanych dokumentów)

| Kolumna | Opis |
|---------|------|
| id | UUID |
| user_id | Kto wygenerował zestaw |
| transport_type | road_tir lub sea |
| country_from | Kraj nadania (np. PL) |
| country_to | Kraj docelowy (np. CN) |
| is_outside_eu | Boolean — czy trasa poza UE |
| cargo_description | Opis towaru |
| cargo_weight_kg | Waga w kilogramach |
| cargo_volume_m3 | Objętość w metrach sześciennych |
| cargo_value | Wartość towaru |
| cargo_currency | Waluta (EUR, CHF, USD, PLN) |
| cargo_packages | Liczba paczek / palet |
| hs_code | Kod celny CN/HS (opcjonalne, dla tras poza UE) |
| sender_company_id | Powiązanie z tabelą companies — nadawca |
| receiver_company_id | Powiązanie z tabelą companies — odbiorca |
| generated_at | Kiedy wygenerowano |
| documents | JSON — lista wygenerowanych dokumentów z linkami do PDF |

### document_templates (szablony dokumentów)

| Kolumna | Opis |
|---------|------|
| id | UUID |
| code | Unikalny kod: cmr, packing_list, invoice, sad, sea_waybill itd. |
| name_pl | Polska nazwa: CMR — list przewozowy |
| name_en | Angielska nazwa: CMR — Consignment Note |
| transport_type | road / sea / both |
| required_for_eu | Boolean — czy wymagany wewnątrz UE |
| required_for_customs | Boolean — czy wymagany dla odprawy celnej |
| template_json | JSON z polami dokumentu i ich mapowaniem |
| version | Numer wersji szablonu |
| updated_at | Kiedy ostatnio aktualizowano szablon |

### payments (płatności)

| Kolumna | Opis |
|---------|------|
| id | UUID |
| user_id | Kto zapłacił |
| stripe_payment_id | ID transakcji w Stripe |
| amount | Kwota w centach (np. 2900 = 29,00 CHF) |
| currency | Waluta (chf, eur, pln) |
| type | monthly_subscription lub pay_per_document |
| status | succeeded / pending / failed |
| created_at | Data transakcji |

## Relacje między tabelami

```
users → document_sets     (1 użytkownik → wiele zestawów dokumentów)
users → companies         (1 użytkownik → wiele zapisanych firm)
document_sets → companies (zestaw ma nadawcę i odbiorcę z tabeli companies)
users → payments          (historia płatności powiązana z kontem)
```

## Ważne zasady

- Pliki PDF nie są przechowywane wiecznie — link do pobrania wygasa po 7 dniach
- Dane firm są szyfrowane w bazie (szczególnie NIP i dane adresowe)
- Zgodność z RODO — użytkownik może zażądać usunięcia wszystkich swoich danych
- Backup bazy: automatyczny co 24h
